# TOWMEI ERP 操作テストシナリオ

システムが意図通りに動作し、ユーザーが快適に操作できるかを確認するためのテスト項目です。

## 1. ダッシュボード (Dashboard)
- [ ] **データの可視化**: チャート（棒グラフ、ライン、エリア）が正しく表示されているか。
- [ ] **納期切迫アラート**: 期限が近い工程が赤字でハイライトされているか。
- [ ] **クイックアクション**: 登録ボタンがスマホでも押しやすいか。
- [ ] **通知確認**: 右上のベルアイコンから通知一覧が開けるか。

## 2. 受注管理 (Orders)
- [ ] **検索機能**: 顧客名や受注番号で正しくフィルタリングされるか。
- [ ] **レスポンシブ表示**: スマホ幅でサイドバーがドロワー（引き出し）形式になり、メニューが開閉できるか。
- [ ] **ステータス表示**: 「一部出荷」などのステータスが色分けで見やすいか。

## 3. 在庫管理 (Inventory)
- [ ] **棚卸操作**: 在庫調整モーダルがスムーズに開き、数値を入力して保存できるか（UI上の挙動）。
- [ ] **履歴確認**: 在庫履歴が時系列で正しく並んでいるか。
- [ ] **フィルタリング**: 商品種別（完成品/原材料）の切り替えが機能するか。

## 4. 支払管理 (Payments)
- [ ] **ダブルチェックフロー**: 承認ボタンや確定ボタンが適切な場所に配置されているか。
- [ ] **明細表示**: 各外注先の明細がアコーディオンで展開・格納できるか。
- [ ] **出力機能**: CSV出力ボタンが反応するか。

## 5. 共通・UX
- [ ] **モバイル操作性**: 指一本で全てのボタンが届くか、文字が小さすぎないか。
- [ ] **読み込み状態**: Skeleton UI（グレーの枠）が正しく表示され、画面のガタつきがないか。
- [ ] **エラー表示**: 入力エラー時に赤いメッセージなどが適切に出るか。
